================================================================================
FEATURES IMPLEMENTED (BOTH ROS1 & ROS2)
================================================================================

1. RECORD BAGS
   - Records sensor data to timestamped files/directories
   - Default: 5 seconds, /scan and /odom topics
   - Customizable duration and topics
   - Automatic timestamp-based naming

2. PLAY BAGS
   - Playback recorded sensor data
   - Configurable playback rate
   - Loop support
   - Start time offset
   - Non-blocking background execution

3. LIST BAGS
   - List all recorded bags
   - Filter by size and pattern
   - Detailed metadata (size, creation time)
   - Sorted by modification time

4. GET BAG INFO
   - Detailed bag information
   - Topics, message counts, duration
   - Start/end times

5. DELETE BAGS
   - Safe deletion with error handling
   - Confirmation messages

6. DIRECTORY MANAGEMENT
   - Automatic bags/ directory creation
   - Centralized storage location

================================================================================
KEY DIFFERENCES: ROS1 vs ROS2
================================================================================

File Format:
  - ROS1: Single .bag file (binary)
  - ROS2: Directory with metadata.yaml and .db3 SQLite database

Commands:
  - ROS1: rosbag record / rosbag play
  - ROS2: ros2 bag record / ros2 bag play

Function Names:
  - ROS1: rosbag_* functions
  - ROS2: ros2bag_* functions

Storage:
  - ROS1: bags/YYYYMMDD_HHMMSS.bag
  - ROS2: bags/YYYYMMDD_HHMMSS/
